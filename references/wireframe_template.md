# Wireframe Template

## 목적

이 템플릿은:

* 1pager
* PRD
* 아이디어 메모

등을 기반으로 빠르게 화면 구조를 탐색하기 위한
텍스트 기반 Wireframe 템플릿입니다.

특히 본 템플릿은:
"정답 Wireframe 1개"가 아니라,
서로 다른 방향성의 여러 UI 컨셉을 동시에 탐색하는 데 목적이 있습니다.

---

# 핵심 원칙

## 하나의 아이디어라도 최소 3개의 다른 방향으로 제안합니다.

동일한 기능/화면이라도:

* 현실적인 구조
* 디자인 강조 구조
* 미래지향적 구조

등 서로 다른 UX 방향성을 함께 제안합니다.

핵심은:
"빠르게 비교하고 아이디어를 발산하는 것"입니다.

---

# 추천 Wireframe 유형

프로젝트 성격에 따라 유형 이름은 변경 가능합니다.

기본적으로 아래 3가지 방향을 추천합니다.

| 유형        | 설명                                |
| --------- | --------------------------------- |
| 현실적인 안    | 현재 서비스 구조와 운영 현실성을 고려한 안정적 구조     |
| 디자인 강조안   | 브랜드 경험, 몰입감, 비주얼 임팩트를 강화한 구조      |
| 실험적/미래형 안 | 새로운 인터랙션, AI, 자동화, 과감한 UX를 적용한 구조 |

---

# Wireframe 작성 원칙

## 1. 구조 중심으로 작성합니다.

중요한 것은:

* 레이아웃
* 정보 우선순위
* 사용자 흐름
* CTA 위치

입니다.

디자인 디테일보다는 구조에 집중합니다.

---

## 2. 실제 서비스처럼 작성합니다.

❌ 단순 박스 나열

✅ 실제 서비스 흐름 고려

* 실제 카드 개수
* 실제 콘텐츠 길이
* 실제 CTA
* 실제 Empty 상태

---

## 3. 화면 목적이 명확해야 합니다.

각 화면은:

* 구매
* 탐색
* 가입
* 저장
* 리텐션

등 하나의 핵심 목적을 가져야 합니다.

---

## 4. 각 안의 차이가 명확해야 합니다.

❌ 거의 비슷한 화면 3개

✅

* 정보 구조 자체가 다름
* CTA 방식 다름
* 인터랙션 방향 다름
* 콘텐츠 강조 방식 다름

---

# 화면 템플릿

# 화면명

## 화면 목적

이 화면의 핵심 목적:
- 탐색
- 구매
- 가입
- 콘텐츠 소비
- 리텐션
- 개인화
등

---

## 공통 요구사항

- 반드시 포함되어야 하는 기능
- 정책상 필요한 영역
- 운영상 필요한 요소
- 데이터/광고/추천 조건

예시:
- 검색창 포함 필수
- 추천영역 최소 2개
- 하단 탭바 유지
- 광고 슬롯 1개 포함

# Type A — 현실적인 안

## 컨셉

현재 서비스 구조를 크게 벗어나지 않으면서:
- 빠르게 적용 가능
- 운영 안정성 높음
- 사용자 학습 부담 낮음

방향으로 설계.

---

## 핵심 UX 방향

- 익숙한 패턴 유지
- CTA 명확화
- 정보 탐색 효율 강화

---

## Wireframe

```text
┌─────────────────────┐
│ Header              │
│ Logo      Search    │
├─────────────────────┤
│ Hero Banner         │
├─────────────────────┤
│ Personalized Cards  │
│ [Card] [Card]       │
├─────────────────────┤
│ Recommended Items   │
├─────────────────────┤
│ Bottom Navigation   │
└─────────────────────┘
````

---

## 특징

* 현재 구조와 유사
* 개발/운영 리스크 낮음
* 빠른 MVP 가능

---

## 장점

* 사용자 적응 쉬움
* 구현 난이도 낮음
* 운영 효율 높음

---

## 단점

* 차별화 약함
* 브랜드 임팩트 제한적

---

# Type B — 디자인 강조안

## 컨셉

브랜드 경험과 몰입감을 강화한 구조.

콘텐츠와 비주얼 중심 UX 적용.

---

## 핵심 UX 방향

* 대형 비주얼
* 감성 중심
* 스크롤 몰입 강화

---

## Wireframe

```text
┌─────────────────────┐
│ Full Hero Visual    │
│ Large CTA           │
├─────────────────────┤
│ Curated Story Cards │
│  Large Image Cards  │
├─────────────────────┤
│ AI Personalized Feed│
├─────────────────────┤
│ Floating Action CTA │
└─────────────────────┘
```

---

## 특징

* 브랜드 몰입감 강화
* 콘텐츠 중심 구조
* 비주얼 우선 설계

---

## 장점

* 차별화 가능
* 마케팅 활용도 높음
* 브랜딩 효과 큼

---

## 단점

* 구현 공수 증가
* 정보 탐색 효율 저하 가능

---

# Type C — 실험적 / 미래형 안

## 컨셉

AI/자동화/실시간 개인화를 적극 활용한 미래형 구조.

기존 앱 패턴을 과감히 벗어남.

---

## 핵심 UX 방향

* AI 중심 인터랙션
* 자동 추천
* 상황 기반 UI 변화

---

## Wireframe

```text
┌─────────────────────┐
│ AI Dynamic Greeting │
│ "오늘 이런 혜택..." │
├─────────────────────┤
│ Adaptive Feed       │
│ 실시간 UI 재배열     │
├─────────────────────┤
│ Conversational CTA  │
│ "추천해줘"           │
├─────────────────────┤
│ Context-aware Cards │
└─────────────────────┘
```

---

## 특징

* AI 기반 구조
* 실시간 개인화
* 동적 인터페이스

---

## 장점

* 미래지향적 경험
* 차별화 강함
* AI 서비스 확장 용이

---

## 단점

* 사용자 학습 필요
* 운영 복잡도 증가
* 구현 난이도 높음

* Default
* Loading
* Empty
* Error
* Disabled

## 인터랙션 설명

가능하면:

* 등장 애니메이션
* 스크롤 변화
* Sticky 동작
* Transition

등도 함께 설명합니다.

---

## 모바일 기준 우선 작성

특별한 이유가 없다면:
모바일 기준으로 작성하는 것을 권장합니다.

---

# 추천 활용 방식

## 아이디어 발산 단계

1pager 기반:

* 다양한 방향 탐색
* UX 가능성 검토

---

## PRD 이전 단계

* 정보 구조 검증
* 사용자 흐름 검증

---

## AI 프로토타이핑 단계

Claude / GPT / Cursor 등에 입력하여:

* UI 생성
* 코드 생성
* Figma 초안 생성

등으로 활용 가능합니다.

---

# 마지막으로

좋은 Wireframe의 핵심은:
"정답을 만드는 것"이 아닙니다.

여러 방향성을:

* 빠르게 비교하고
* 토론하고
* 발전시키는 것

이 가장 중요합니다.

특히 AI 시대의 Wireframe은:
최종 설계도라기보다,
아이디어 탐색과 UX 사고 확장을 위한 도구에 가까워지고 있습니다.
