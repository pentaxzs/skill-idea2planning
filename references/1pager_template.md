# 템플릿
## 1. 한 줄 요약
### 제안하고자 하는 내용을 해주세요. 한 두 줄로 내로 짧게 설명
예: 본 과제는 5060 사용자 신규 유입 증대를 목표로 하는 로그인 절차 간소화에 대해 다룹니다.

## 2. 대상
### 어떤 고객을 대상으로 한 제안인가요?
예: 당사앱을 사용하지 않는 5060 사용자

## 3. 문제 정의
### 해결하고 싶은 고객 / 비즈니스의 문제 는 무엇인가요? 이것이 문제임을 어떤 데이터를 통해 알 수 있었나요?
해결하고 싶은 문제 와 그것이 문제임을 파악할 수 있는 현황데이터를 기재합니다. 단 현황 데이터 측정이 어렵거나 현 상황에서 불가능 하다면, 정성적 근거를 적어주셔도 무방합니다.
예: 업주의 마케팅 수단이 현저히 부족하다. - 고객센터 및 영업 담당자를 통해 인입되는 상당수(전체 부정 의견 중 N%, 월 평균 N건)가 마케팅 수단 부재에 대한 내용
### 문제가 발생하는 근본적인 원인은 무엇인가요? 
예: 20년 4월 1일, 업주의 유일한 마케팅 수단인 단골쿠폰 상품이 F/O되었다.
(특정한 고객 / 비즈니스의 문제를 해결하기 위한 과제가 아닌 경우, 아래 항목을 채워주세요.)
### (선택) 제안하는 내용 통해 검증하고 싶은 가설은 무엇인가요?
예: GPT 기술을 기반으로 브랜드 소개, 상품 설명 작성에 드는 업주 공수를 줄인다면, 브랜드 관리 소요시간이 절감되어 브랜드 만족도가 높아질 것이다.

## 4. 목표
### 위에서 정의한 문제를 성공적으로 해결했다고 할 수 있으려면, 어떤 목표를 달성해야 하나요? 혹은 성공적으로 해결 한다면, 어떤 임팩트를 만들어 낼 수 있나요?
목표의 수준이 담긴 측정 가능한 지표를 기재합니다.
목표 지표와 가드레일 지표로 구성될 수 있습니다. 
가능한 정량적 / 검증 가능한 수치로 기재해주세요.
#### 목표 지표: 본 작업을 통해 달성해야 하는 주요 지표
예: 3번 탭의 ‘장바구니' 추천 영역 변경안 클릭율 N% 달성
#### 가드레일 지표: 본 작업으로 인해 떨어져서는 안되는 주요 지표
예: 3번 탭의 ‘장바구니' 추천 영역 매출

## 5. 현황
#### (선택) 본 제안와 관련하여 현재 논의 중인/ 과거에 논의했던 이력이 있나요?
본 과제를 현재 / 과거에 논의한 바 있다면 논의 상태 및 히스토리를 기재합니다.
#### 해결방안을 도출해내는 데에 앞서, 정의한 문제와 관련된 정보나 분석 내용이 있나요? 
내 / 외부 상황, 기존 프로덕트 / 비즈니스 데이터와 정책 등, 정의한 문제와 관련된 정보와 분석 내용들을 기재합니다.
예: 검색결과 기본순 로직 변경 > 현 검색결과 목록 기본순 정책 ( 참고 )
* 검색 연관도 점수, 검색히스토리 점수 등 각 항목을 항목 별 로직으로 환산한 score를 총합하여 매장의 total score 산정
* 1-10위는 상위 결과로 구성되어 있으며, 앱에서 새로고침시 랜덤으로 정렬
* 나머지는 랜덤없이 랭킹 점수대로 정렬
예: 구매리스트 내 재구매 기능 추가 > 재구매 현황 분석 ( 참고 )
* 최근 3개월 내 동일 브랜드에서 1회 이상 구매한 경험이 있는 사용자는 전체 중 N건으로 전체의 N% 수준
* 이들이 3개월 내 발생시킨 구매는 N건이며, 이는 전체 구매의 N%를 차지
* 한 브랜드 내 누적 구매가 N건 이상인 경우가 구매건수 기준 N건으로 전체 구매의 N%를 차지
→ 재구매 타깃 사용자 층이 충분히 넓으며, 이들이 발생시키고 있는 구매가 전체 중 대다수 → 타깃 사용자에게 구매편의성 제고 기대

## 6. 해결방안
#### 문제 해결을 위한 주요 방안은 무엇인가요? 
정의한 문제와 목표에 벗어나지 않는기능 / 해결방안 관련된 사례 / 개발 요구사항 등을 핵심만 간결하게 기재합니다. 기능별 우선순위가 있다면 기재하고, 그렇게 판단한 근거를 포함합니다.
예: 구매내역 내 재구매 기능 추가( 참고 )
* (우선순위 1) 재구매하기 기능이 장바구니 목록/상세에서 다른 기능에 시각적으로 주목도가 있어야 한다.
  * 현재 장바구니 목록/상세에 노출되는 버튼(리뷰쓰기 등)과 다른 UI로 구성되어야 함
  * 고객이 장바구니 발견 후 '재구매하기' 기능을 제일 먼저 발견할 수 있어야 함
* (우선순위 1)재구매하기 팝업이 열리고 닫히는 것이 마치 네이티브 화면처럼 자연스러워야 한다.
  * 팝업이 여닫히는 액션이 부자연스럽지 않아야 여러 구매의 재구매 기능을 자유롭게 눌러 볼 수 있음
  * 팝업이 열렸다가 닫히는 액션으로 인해 화면이 깜박이는 현상 등이 발생하지 않아야 함
* (우선순위 2)재구매 의사가 높은 구매 대부분에 재구매 가능한 상태로 버튼을 보여줄 수 있어야 한다.
  * 브랜드/상품 정보가 변경되는 주기보다 긴 시간 경과된 구매에는 재구매하기 기능을 보여주지 않음
  * 이로 인해 재구매하기 기능이 긴 시간 경과된 것에 모두 비활성화된 상태로 노출되는 것을 방지함

## 7. 리스크
#### 목표 달성을 방해할 수 있는 리스크 또는 감수해야 하는 트레이드 오프(Trade-off) 는 무엇이 있나요? 이를  최소화할 수 있는 방법은 무엇인가요?
예: 개편 시 혜택 영역이 사라져, 고객 커뮤니케이션 어려워지는 점에 대한 브랜드 불만 예상 → 별도 브랜드 커뮤니케이션 영역 신설하여 헷징 ( 참고 )
* 참고: 프로젝트 오픈에 따른 내/외부 리스크 및 법무적 리스크에 대한 포괄적 검토는 리스크 관리 부서에 검토 요청
