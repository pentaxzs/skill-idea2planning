# [26.05.17] 개인화 혜택 모드 자동 전환 제안 (디지털서비스팀)

---

# 1. 프로젝트 한 줄 요약

본 프로젝트는 고객의 혜택 탐색 효율 및 앱 재방문율 향상을 위해
고객의 소비/위치/상황 데이터를 기반으로 앱의 혜택 모드를 자동 전환하는 기능을 제공하는 프로젝트입니다.

---

# 2. 대상

| 대상       | 상황/특징                          |
| -------- | ------------------------------ |
| 고객       | 혜택이 많지만 자신에게 맞는 혜택을 찾기 어려운 사용자 |
| 해외 방문 고객 | 여행 중 필요한 혜택을 즉시 확인하고 싶은 사용자    |
| 라이트 유저   | 월 1~2회만 앱을 사용하는 사용자            |

---

# 3. 문제 정의

## 1) 해결하고 싶은 고객 / 비즈니스 문제는 무엇인가요?

### 고객 문제

* 혜택 종류가 많아 자신과 관련된 혜택을 찾기 어려움
* 해외/쇼핑/주유 등 상황별 혜택을 즉시 확인하기 어려움
* 앱 첫 화면 정보가 반복적으로 느껴짐

### 비즈니스 문제

* 개인화 영역 CTR 감소
* 혜택 사용률 감소
* MAU 정체
* 특정 혜택 캠페인의 낮은 노출 효율

---

## 2) 이것이 실제 문제라는 것을 어떤 데이터로 확인했나요?

| 항목          | 현재 수치    | 인사이트           |
| ----------- | -------- | -------------- |
| 혜택 영역 CTR   | 1.9%     | 최근 6개월 지속 감소   |
| 개인화 메시지 클릭률 | -18% YoY | 반복 노출 피로 추정    |
| 해외 혜택 진입률   | 8%       | 해외 결제 고객 대비 낮음 |
| 앱 월평균 방문수   | 3.1회     | 라이트 유저 증가      |

### 추가 인사이트

* 해외 결제 발생 후 48시간 내 혜택 페이지 진입률 급증
* 공항 인근 위치 기반 진입 고객의 클릭률 2.4배 높음
* 특정 고객군은 동일한 혜택 메시지를 월 12회 이상 반복 노출받음

---

## 3) 문제가 발생하는 근본 원인은 무엇인가요?

### Case 1.

고객 상황 변화와 앱 화면이 연동되지 않음

### Case 2.

혜택 구조가 카드 중심으로 설계되어 사용 맥락 반영이 어려움

### Case 3.

고객 행동 기반 실시간 전환 로직 부재

---

## 4) 왜 지금 해결할 가치가 있나요?

### 기대효과

* 상황 기반 혜택 소비 증가
* 해외/쇼핑/주유 등 특화 모드 활성화
* MAU 및 체류시간 증가

### 경쟁 상황

* 글로벌 금융앱은 위치/행동 기반 UI 전환 확대 중
* 일부 간편결제 서비스는 해외 모드 자동화 제공 시작

---

# 4. 목표

## 핵심 KPI

| 항목        | 목표   | 기준          |
| --------- | ---- | ----------- |
| 혜택 영역 CTR | +25% | 실험군 기준      |
| 해외 혜택 진입률 | +40% | 해외 결제 고객 기준 |
| 월 방문수     | +12% | 전체 고객 평균    |
| 재방문율      | +8%  | 라이트 유저 기준   |

---

# 5. 현황 및 분석

## VOC

* “혜택이 너무 많아서 뭘 봐야 할지 모르겠다”
* “해외 와서 보니 관련 혜택을 뒤늦게 발견했다”

## 리서치

* 고객은 상황 맞춤형 혜택을 더 가치 있게 인식
* 동일 메시지 반복 시 클릭률 급감

---

# 6. 해결방안

## Phase 1. 모드 자동 전환

### 제공 기능

* 해외 모드 자동 진입
* 쇼핑 집중 모드
* 주유/차량 모드
* 주말 라이프 모드

### 예상 효과

* 탐색 시간 감소
* 클릭률 상승

---

## Phase 2. 개인화 UI 고도화

### 제공 기능

* 고객 행동 기반 홈 구조 변경
* 실시간 혜택 우선순위 조정
* AI 추천 카드 노출
