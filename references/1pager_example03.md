# [26.05.17] 콘텐츠 발견 강화 및 AI 추천 피드 제안 (콘텐츠플랫폼팀)

---

# 1. 프로젝트 한 줄 요약

본 프로젝트는 콘텐츠 소비량 및 재방문율 향상을 위해
사용자 관심사 기반 AI 추천 피드를 제공하는 프로젝트입니다.

---

# 2. 대상

| 대상     | 상황/특징              |
| ------ | ------------------ |
| 일반 사용자 | 무엇을 봐야 할지 고민하는 사용자 |
| 신규 사용자 | 취향 데이터가 부족한 사용자    |
| 크리에이터  | 콘텐츠 노출 확대를 원하는 제작자 |

---

# 3. 문제 정의

## 고객 문제

* 콘텐츠 탐색 시간이 길어짐
* 인기 콘텐츠 편중 심화
* 취향 맞춤 추천 부족

## 비즈니스 문제

* 콘텐츠 소비량 정체
* 재방문 감소
* 신규 크리에이터 성장 제한

---

## 데이터 기반 현황

| 항목         | 현재 수치 | 인사이트       |
| ---------- | ----- | ---------- |
| 평균 탐색 시간   | 4.8분  | 소비 전 피로 증가 |
| 신규 콘텐츠 클릭률 | 0.9%  | 상위 콘텐츠 편중  |
| 재방문율       | 28%   | 추천 만족도 낮음  |
| 콘텐츠 완주율    | 37%   | 추천 정확도 부족  |

---

## Root Cause

### Case 1.

추천 알고리즘이 인기 기반 중심

### Case 2.

사용자 감정/상황 반영 부족

### Case 3.

새로운 콘텐츠 발견 구조 부족

---

# 4. 목표

| 항목         | 목표   |
| ---------- | ---- |
| 콘텐츠 클릭률    | +30% |
| 완주율        | +18% |
| 재방문율       | +15% |
| 신규 콘텐츠 소비량 | +40% |

---

# 5. 현황 및 분석

## 사용자 인터뷰

* “늘 보던 것만 보게 된다”
* “새로운 콘텐츠 찾기가 어렵다”

## 시장 동향

* 숏폼 기반 추천 소비 증가
* AI 큐레이션 서비스 확대

---

# 6. 해결방안

## Phase 1. 관심사 기반 추천

### 제공 기능

* 감정/상황 태그 추천
* 시간대별 추천 변경
* 취향 군집 기반 추천

---

## Phase 2. 생성형 AI 큐레이션

### 제공 기능

* “오늘의 추천”
* AI 요약 기반 추천
* 콘텐츠 연결 추천

---

# 7. 리스크

| 영향도    | 리스크          | 대응 방안       |
| ------ | ------------ | ----------- |
| High   | 필터버블 강화      | 탐색 콘텐츠 혼합   |
| Medium | 추천 피로감       | 다양성 알고리즘 추가 |
| Low    | 신규 콘텐츠 품질 편차 | 품질 점수 반영    |

---

# 8. 중요도 및 긴급도

## 중요도 : 상

* 콘텐츠 소비 성장 핵심 과제

## 긴급도 : 중

* 기존 추천 구조 유지 가능하나 성장 한계 존재
